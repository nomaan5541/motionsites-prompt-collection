# Security Policy

## 🔒 Supported Versions

| Version | Supported |
| ------- | --------- |
| Latest  | ✅ Yes    |
| Older   | ❌ No     |

---

## 🚨 Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly:

1. **DO NOT** open a public issue for security vulnerabilities
2. Instead, open a **private security advisory** on GitHub:
   - Go to the repository's **Security** tab
   - Click **"Report a vulnerability"**
3. Or contact us via GitHub Issues with **[SECURITY]** in the title

### Please include:
- Description of the vulnerability
- Steps to reproduce
- Possible impact
- Suggested fix (if any)

---

## ⏱️ Response Time

We aim to:
- **Acknowledge** reports within **48 hours**
- **Assess** severity within **72 hours**
- **Resolve** critical issues as quickly as possible

---

## 🛡️ Security Measures

This project implements the following security measures:
- Content Security Policy (CSP) headers
- X-Content-Type-Options: nosniff
- X-Frame-Options configured per deployment needs
- Regular review of dependencies
- No collection of user data or credentials

---

## 🙏 Responsible Disclosure

Please do not publicly disclose vulnerabilities until they are fixed. We appreciate your help in keeping this project and its users secure.

---

Thank you for helping keep this project secure 🔐
